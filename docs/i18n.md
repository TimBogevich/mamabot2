# Локализация (i18n)

## Обзор

Файлы локализации находятся в директории `functions/src/locales/` и содержат все пользовательские строки для MamaBot Telegram бота. Каждый файл представляет один язык и имеет идентичную структуру ключей.

Языковые коды (`ru`, `en`) соответствуют полю `language` в коллекции `users` (`'ru' | 'en'`).

Дублирующаяся копия находится в `functions/src/i18n/` (для тестов паритета ключей).

---

## Файлы

| Файл | Язык |
|------|------|
| `functions/src/locales/ru.json` | Русский |
| `functions/src/locales/en.json` | Английский |

## Структура ключей

Ключи используют dot-нотацию с сегментами в snake_case. Вложенные объекты разрешаются через точку, например:

```
onboarding.welcome         → "👋 Добро пожаловать..."
error.generic              → "❌ Произошла ошибка..."
```

### Топ-уровневые категории

| Категория           | Описание                              |
|---------------------|---------------------------------------|
| `onboarding`        | Экран приветствия, регистрации и ввода дат |
| `menu`              | Главное меню и кнопки навигации       |
| `settings`          | Меню настроек                         |
| `error`             | Сообщения об ошибках                  |
| `notifications`     | Уведомления о неделях беременности    |
| `commands`          | Описания слеш-команд бота             |
| `help`              | Текст справки                         |
| `week`              | Информация о неделе беременности (сводка, детальные секции, навигация) |
| `mood`              | Дневник настроения                    |
| `nutrition`         | Питание                              |
| `partner`           | Приглашение партнёра                 |



Строки могут содержать плейсхолдеры в фигурных скобках:

| Плейсхолдер | Описание                         | Пример                           |
|-------------|----------------------------------|----------------------------------|
| `{week}`    | Номер недели беременности        | `"{week} неделя беременности"`   |
| `{date}`    | Дата в формате YYYY-MM-DD        | `"Дата сохранена: {date}"`       |
| `{code}`    | Код-приглашение для партнёра     | `"Твой код: {code}"`             |
| `{size}`    | Размер плода (сравнение)         | `"📏 Размер: {size}"`             |
| `{weight}`  | Вес плода в граммах              | `"⚖️ Вес: {weight} г"`            |

## Конвенция именования ключей

1. **Все ключи в snake_case** — например, `baby_development`, `meal_types`, `vitamin_recommendations`.
2. **Топ-уровневые категории** — существительные в единственном числе: `menu`, `mood`, `nutrition`.
3. **Вложенные объекты** — для группировки связанных ключей: `meal_types.breakfast`, `meal_types.lunch`.
4. **Единообразие** — одинаковые ключи в обоих языковых файлах; каждый ключ присутствует в `ru.json` и `en.json`.

---

## Как добавить новый ключ

1. Добавьте ключ в `ru.json` с русским переводом.
2. Добавьте **точно такой же ключ** в `en.json` с английским переводом.
3. Если ключ относится к новой категории, создайте новый топ-уровневый объект.
4. Запустите тесты для проверки паритета ключей:
   ```bash
   cd functions
   npx vitest run src/i18n/
   ```
5. Тесты проверят:
   - Оба файла существуют и содержат валидный JSON
   - Все ключи (включая вложенные) идентичны в обоих файлах
   - Нет пустых строк
   - `ru.json` содержит кириллические символы

## Использование

Файлы загружаются через `require()` в модуле `i18n.js`:

```js
const ru = require('./locales/ru.json');
const en = require('./locales/en.json');
```

Функция `t(chatId, key, params)` доступна через:

```js
const { t } = require('./i18n');
const text = await t(chatId, 'onboarding.welcome', { name: 'Анна' });
```

## Тестирование

Тесты находятся в `functions/src/i18n/__tests__/locales.test.js` и используют Vitest.

```bash
cd functions
npx vitest run src/i18n/
```

Тесты покрывают:
- Валидность JSON
- Паритет структуры ключей (рекурсивно)
- Отсутствие пустых значений
- Наличие кириллицы в русском файле
- Наличие обязательных категорий
